# Contributing to Awesome AI API Gateways

Thanks for thinking about contributing.

## Quick rules

1. **One service per PR.** Easier to review and discuss.
2. **No paid promotion or unsolicited self-listings.** Listings exist to help readers, not to acquire traffic.
3. **Disclose affiliations.** If you work for the service you're proposing, say so in the PR description. We'll still review on merit, but openness is required.
4. **Working URLs.** Broken or parked URLs will be removed.
5. **Be honest about tradeoffs.** Mention weaknesses (high fees, limited models, fragile uptime) where applicable.
6. **Follow existing format.** Column conventions for tables, sentence-case in descriptions, no emoji spam.

## What we accept

- **Hosted AI API gateways** (any payment model)
- **Self-hosted / open-source AI gateways**
- **Pricing or latency comparison tools** that are vendor-neutral
- **High-quality articles or research** on gateway design, AI economics, geographic access, crypto payment for AI

## What we don't accept

- Pure model providers that lack a gateway/proxy use-case
- Services without a working public API or signup flow
- Listings that exist solely to drive traffic without adding catalogue value
- Anything that claims "0 fees" but charges hidden surcharges
- Services known to be scams, rugged, or that materially misrepresent offerings

## Submission format

For service entries, use:

```markdown
| [Service Name](https://service-url.com) | 30+ | 5% | USDT (TRC20) | None | $10 |
```

For article entries:

```markdown
- [Article Title](https://url) — One-sentence description. *(Publisher / Author)*
```

## PR description template

```
**What:** [Short description of what you're adding]
**Why it belongs:** [Why is this useful for readers comparing gateways]
**Affiliation:** [None / I work at $service / Disclosure]
**Verification:** [Link to public pricing page or docs confirming the entry's facts]
```

## Review timeline

Stormfire reviews PRs weekly. Expect 3-7 days for an initial response.

## Maintainer

[Stormfire](https://stormfire.io) maintains this list. We add our own service to the list and clearly mark it; we also list our direct competitors fairly. If you spot a fairness issue, open an issue and we'll address it publicly.

## Code of Conduct

Treat each other with respect. We don't tolerate harassment, doxxing, or bad-faith argumentation. Maintainers reserve the right to close PRs and lock issues that violate this.
